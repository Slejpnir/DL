-1) Безкоштовний запуск

  • Google Colab: достатньо акаунта; можливо отримати GPU. Підійде й CPU.
  • Локально: Python ≥ 3.9, pip install ultralytics. Перевірка: yolo checks.

0) Передумови та очікувані артефакти

  • Набір власних зображень (від 50–300) по темі; бажано різноманітні сцени й ракурси.
  • Під час здачі: скрипт/ноутбук, dataset.yaml, метрики, 3–5 прикладів інференсу на нових зображеннях, за бажання — експорт у ONNX.

1) Структура датасету у форматі YOLO

dataset/
  images/
    train/   # *.jpg|*.png
    val/
    test/    # опційно
  labels/
    train/   # *.txt — один рядок на об’єкт: class x_center y_center width height (усе в [0,1])
    val/
    test/
dataset.yaml

Файли анотацій .txt з таким самим базовим ім’ям, як у відповідного зображення (наприклад, img001.jpg → img001.txt).

2) Анотація власних даних у Label Studio (BBox)

  1. Встановіть Label Studio (локально або docker) та запустіть інтерфейс.
  2. Створіть проєкт «Object Detection» з одним або кількома класами (наприклад, <RectangleLabels> з категоріями).
  3. Імпортуйте свої зображення, розмічайте прямокутниками (BBox).
  4. Експорт: у менеджері даних оберіть Export → формат YOLO (Ultralytics) або, якщо недоступно, експортуйте в JSON і сконвертуйте в YOLO-конвертором.
  5. Розпакуйте експорт у структуру, наведену вище (images/ та labels/ для кожного спліту).

За потреби можна використати CVAT: експортуйте у форматі Ultralytics YOLO (Detection) і розкладіть файли за сплітами.

3) dataset.yaml — приклад і валідність

# dataset.yaml (приклад)
path: /content/dataset           # кореневий шлях до датасету
train: images/train
val: images/val
test: images/test                # опційно

names:
  0: клас_0
  1: клас_1
  # 2: клас_2 ...

Переконайтесь, що names відповідають індексам класів у ваших .txt. Шляхи мають існувати.

4) Підготовка середовища

Colab

!pip install -q ultralytics
from ultralytics import YOLO
import torch; print("CUDA:", torch.cuda.is_available())

Локально

pip install ultralytics
yolo checks

5) «Саніті-чек» на COCO8 (опційно, 2–3 хв)

# Train
yolo train model=yolo11n.pt data=coco8.yaml epochs=3 imgsz=640
# Predict
yolo predict model=yolo11n.pt source=path/to/image.jpg

Це дозволяє швидко перевірити працездатність пайплайна перед навчанням на своїх даних.

6) Навчання на своїх даних (train)

# CLI
yolo train model=yolo11n.pt data=dataset.yaml epochs=30 imgsz=640 batch=16 patience=10 device=auto

# Python
from ultralytics import YOLO
model = YOLO("yolo11n.pt")
model.train(data="dataset.yaml", epochs=30, imgsz=640, batch=16, patience=10, device="auto")

Результати (ваги, графіки) зберігаються у runs/detect/train*.

7) Валідація та метрики (val)

yolo val model=runs/detect/train/weights/best.pt data=dataset.yaml

Зніміть mAP@0.50, mAP@0.50:0.95, Precision/Recall, PR-криві, Confusion Matrix — додайте у звіт.

8) Обов’язкове тестування на власних зображеннях (predict)

# Підготуйте нові (раніше не бачені) зображення
mkdir -p my_samples
# ...покладіть 5–10 зображень всередину...

# Інференс і збереження візуалізацій
yolo predict model=runs/detect/train/weights/best.pt source=my_samples/ conf=0.25 save=True

Додайте до звіту 3–5 прикладів зображень із намальованими боксами та впевненістю.

9) Тюнінг і підсилення якості

  • Аугментації за замовчуванням увімкнено; можна збільшити епохи, batch, налаштувати LR.
  • Спробуйте інші ваги: yolo11s/m/l/x.pt — компроміс точність/швидкість.
  • Баланс класів і якість розмітки суттєво впливають на mAP.

10) Експорт моделі

# приклад у ONNX
yolo export model=runs/detect/train/weights/best.pt format=onnx

# далі можна виконати інференс і на експортованій моделі
yolo predict model=runs/detect/train/weights/best.onnx source=my_samples/
